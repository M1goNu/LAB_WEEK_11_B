1. When a user takes a picture, that picture is stored in a path based on the given URI. In which part of the code handles this? (Copy that part of the code as the answer) 
Penyimpanan file foto ditangani dalam fungsi insertToStore() di file ProviderFileManager.kt, dengan kode seperti berikut:
private fun insertToStore(fileInfo: FileInfo, contentUri: Uri, contentValues: ContentValues) {
    executor.execute {
        val insertedUri = contentResolver.insert(contentUri, contentValues)
        insertedUri?.let {
            val inputStream = contentResolver.openInputStream(fileInfo.uri)
            val outputStream = contentResolver.openOutputStream(insertedUri)
            IOUtils.copy(inputStream, outputStream)
        }
    }
}

2. In your FileInfo.kt, there are 5 attributes. On the first attribute, what does the URI refer to? And on the fourth attribute, what does relativePath refer to? 
Atribut pertama pada FileInfo.kt adalah uri. Uri merupakan URI sementara untuk membuat file foto/video sebelum disimpan ke MediaStore. URI digunakan kamera untuk menulis hasil foto ke file internal aplikasi, contoh:
content://com.example.lab_week_11_b.camera/my_images/img_12345.jpg
RelativePath merupakan folder tujuan di MediaStore untuk menyimpan file.
Dengan contoh untuk foto tempat penyimpanannya "pictures" dan untuk video "Movies". Nilai akan digunakan Ketika MediaStore menempatkan file ke dalam direktori public seperti: Internal Storage / Pictures.
3. [Bonus] Explain the chronological order from when a user takes a picture until the file is stored in the MediaStore. 
Tahapan pertama Ketika user menekan tombol foto, maka MainActivity akan menangani tombol foto dengan fungsi openImageCapture(). Kemudian aplikasi membuat FileInfo dalam kode generatePhotoUri() yang membuat file kosong, URI sementara via FileProvider, nama file, relativePath, dan mimeType. Lalu kamera dipanggil melalui ActivityReultContracts.TakePicture() yang akan menulis hasil foto ke file berdasarkan URI sementara. Setelah foto berhasil, akan memanggil fungsi insertImageToStore(), fungsi ini akan menyalin file ke MediaStore. Terakhir foto akan tersimpan permanen didalam MediaStore dan dapat diakses oleh aplikasi Gallery
